# Community Engagement Guide

This document outlines how to engage with the Home Assistant community for the Tuya BLE Mesh integration.

## Communication Channels

### Home Assistant Community Forum
- **Category**: Integrations
- **Tags**: `tuya`, `ble`, `bluetooth`, `mesh`
- **URL**: https://community.home-assistant.io/c/configuration/integrations

### GitHub
- **Repository**: https://github.com/home-assistant/core
- **Issue Labels**: `integration: tuya_ble_mesh`
- **Discussions**: https://github.com/home-assistant/core/discussions

### Discord
- **Server**: Home Assistant Discord
- **Channel**: `#integrations`

## Engagement Activities

### 1. Release Announcements
When releasing new versions or features:
- Post in HA Community Forum
- Create GitHub Discussion
- Announce in Discord `#integrations`
- Include: version, key features, breaking changes, upgrade instructions

**Template:**
```
🚀 Tuya BLE Mesh Integration v{VERSION} Released

What's New:
- Feature 1
- Feature 2
- Bug fix 3

Breaking Changes:
- None / List changes

Upgrade Instructions:
1. Update via HACS or manual install
2. Restart Home Assistant
3. Re-pair devices if needed

Feedback welcome!
```

### 2. Feature Requests & Bug Reports
Monitor and respond to:
- GitHub Issues with `integration: tuya_ble_mesh` label
- Forum posts tagged `tuya` + `ble`
- Discord mentions

**Response Template:**
```
Thanks for reporting! Can you provide:
- Home Assistant version
- Integration version
- Device model
- Logs (enable debug: `logger.integration.tuya_ble_mesh: debug`)
- Steps to reproduce
```

### 3. Community Support
Provide support via:
- Answering forum questions
- Helping troubleshoot in Discord
- Commenting on GitHub issues

**Support Guidelines:**
- Be respectful and patient
- Ask for logs and debug info
- Link to documentation
- Escalate complex issues to GitHub

### 4. Documentation
Maintain up-to-date docs:
- `README.md` - Installation, setup, features
- `CHANGELOG.md` - Version history
- `CONTRIBUTING.md` - Development guide
- `COMMUNITY.md` - This file

### 5. Showcase Success Stories
Encourage users to share:
- Working device configs
- Automation examples
- Success stories

Highlight in:
- GitHub Discussions
- Forum posts
- README examples section

## Community Guidelines

### Communication Style
- Be friendly and welcoming
- Avoid jargon when possible
- Provide clear examples
- Link to documentation

### Response Times
- **Critical bugs**: Within 24 hours
- **Feature requests**: Within 1 week
- **Questions**: Within 3 days
- **PRs**: Review within 1 week

### Conflict Resolution
If disagreements arise:
1. Stay calm and professional
2. Focus on technical merits
3. Seek compromise
4. Escalate to HA core team if needed

## Contributing

Encourage community contributions:
- Device compatibility reports
- Bug fixes
- Feature implementations
- Documentation improvements
- Translations

Link to `CONTRIBUTING.md` for details.

## Recognition

Thank contributors:
- Mention in CHANGELOG
- GitHub stars for helpful issues
- Forum badges/likes
- Co-authored commits

## Metrics to Track

Monitor community health:
- GitHub stars/forks
- Forum post views/replies
- Discord mentions
- Issue open/close rate
- PR merge rate
- Active contributors
